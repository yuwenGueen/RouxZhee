# ✅ 任务语法示例

本文档详细介绍RouxZhee中支持的所有任务列表语法特性，包含完整的语法示例和效果展示。

> 💡 **提示**：本文档中的所有语法示例均已在RouxZhee中测试通过，可直接复制使用。

---

## 🎯 基础任务列表

### 1️⃣ 未完成任务

**语法**：
```markdown
- [ ] 未完成的任务项
- [ ] 另一个未完成的任务
```

**效果**：
- [ ] 未完成的任务项
- [ ] 另一个未完成的任务

---

### 2️⃣ 已完成任务

**语法**：
```markdown
- [x] 已完成的任务项
- [x] 另一个已完成的任务
```

**效果**：
- [x] 已完成的任务项
- [x] 另一个已完成的任务

---

### 3️⃣ 混合任务列表

**语法**：
```markdown
- [x] 已完成任务
- [ ] 未完成任务
```

**效果**：
- [x] 已完成任务
- [ ] 未完成任务

---

## 🎨 嵌套任务列表

### 4️⃣ 多级任务列表

**语法**：
```markdown
- [x] 主要任务 1
  - [x] 子任务 1.1
  - [ ] 子任务 1.2
- [ ] 主要任务 2
```

**效果**：
- [x] 主要任务 1
  - [x] 子任务 1.1
  - [ ] 子任务 1.2
- [ ] 主要任务 2

---

## 🌟 高级用法

### 5️⃣ 任务列表与其他元素结合

**语法**：
```markdown
- [x] [查看 Vue 文档](https://vuejs.org)
- [x] 配置 `vite.config.ts`
- [x] **重要**：修复安全漏洞
```

**效果**：
- [x] [查看 Vue 文档](https://vuejs.org)
- [x] 配置 `vite.config.ts`
- [x] **重要**：修复安全漏洞

---

### 6️⃣ 任务列表中的引用

**语法**：
```markdown
- [x] 完成任务 A
  > 备注：这个任务比较简单
- [ ] 完成任务 B
  > 注意：需要先完成依赖任务
```

**效果**：
- [x] 完成任务 A
  > 备注：这个任务比较简单
- [ ] 完成任务 B
  > 注意：需要先完成依赖任务

---

## 💡 最佳实践

### 1️⃣ 任务粒度控制

```markdown
<!-- 好的示例 - 任务粒度适中 -->
- [ ] 完成用户登录功能
  - [ ] 设计登录页面 UI
  - [ ] 实现登录接口调用

<!-- 不好的示例 - 任务过于笼统 -->
- [ ] 完成整个项目
```

### 2️⃣ 任务状态管理

```markdown
- [ ] 🆕 新任务：添加用户注册功能
- [x] ✅ 已完成：修复导航栏样式
```

**效果**：
- [ ] 🆕 新任务：添加用户注册功能
- [x] ✅ 已完成：修复导航栏样式

### 3️⃣ 优先级标记

```markdown
- [ ] 🔴 高优先级：修复生产环境 Bug
- [ ] 🟡 中优先级：优化页面性能
```

**效果**：
- [ ] 🔴 高优先级：修复生产环境 Bug
- [ ] 🟡 中优先级：优化页面性能
